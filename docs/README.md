## Objectif

L'objectif de cette documentation est de donner une vue claire et exploitable du produit:

- le besoin metier,
- les comportements attendus,
- l'architecture technique,
- la structure des donnees,
- et les regles de conduite du projet.

Ces documents sont pensés pour etre lisibles par:

- le metier,
- le dev,
- la QA,
- et toute personne qui reprend le projet.
