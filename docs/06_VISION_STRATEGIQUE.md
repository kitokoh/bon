# BON - Vision strategique

## 1. Ambition

BON n'est pas seulement un script d'automatisation. L'ambition est de faire de ce projet une petite plateforme operationnelle locale, maintenable, documentee et transmissible.

## 2. Vision

La vision du projet repose sur six piliers:

- des robots isoles par compte,
- une base SQLite comme source de verite,
- des contenus preenregistres et reutilisables,
- un menu interactif simple pour les actions courantes,
- une supervision locale suffisante pour piloter,
- et une documentation qui permet de reprendre sans perdre le contexte.

## 3. Resultat attendu

L'utilisateur doit pouvoir:

- lancer l'app sans memoire technique complexe,
- choisir une action en quelques secondes,
- retrouver les groupes et textes deja preparés,
- et observer un comportement coherent entre les executions.

## 4. Positionnement

Le projet est positionne comme:

- un outil de production locale,
- un outil de pilotage de robots,
- un outil de test et de reprise,
- et une base technique evolutive.

Il n'est pas positionne comme:

- une simple demo,
- un prototype jetable,
- ou une configuration a ressaisir a chaque execution.

## 5. Priorites long terme

### Priorite 1

Stabiliser la base de donnees et les donnees de test.

### Priorite 2

Garder les parcours usuels lisibles et rapides.

### Priorite 3

Conserver une documentation au niveau du code.

### Priorite 4

Faciliter la reprise, les tests et les evolutions.

## 6. Indicateurs de reussite

- Menu interactif clair.
- Robot de test preconfigure.
- Groupes visibles sans ressaisie.
- Textes reutilisables depuis SQLite.
- Reprise possible avec les seuls documents.
